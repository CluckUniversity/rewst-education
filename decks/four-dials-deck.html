04 / 06
Strategy + skills

Where to start, and when to write it down

1 · Picking a starting model
Step down Start at the most capable model. Step down until quality drops. Use when: the task repeats and you can measure quality against a fixed bar. Example: coding.
Escalate Start where you'd normally land. Move up only if the result falls short. Use when: it's a one-off, ad hoc request.

The decider is the shape of the task, not the subject. Repeats with a measurable bar → step down. Single ask → escalate. Technical vs non-technical doesn't matter.

2 · Is it worth a skill?
Frequent, but the process is still shifting Not ready to encode
Rare, but high stakes to get right Worth encoding

Author at max, run lean. Build the skill with the most capable model at high effort — every future run inherits that judgment. Once it's solid, the runs can drop down.
